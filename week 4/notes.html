Object oriented programming

In Object-oriented Programmming there are three main concepts. 1. Encapsulation - Parts of the program are inside objects and only essential functionalities are exposed to the end user. 2. Polymorphism - The same process can be used for different objects. Various objects can be used for the same method. 3. Inheritance - You can take an object that already exists and Inherit all its properties and methods. A real world example I can think of is a lamp. There's a lot to a lamp but all we see is a simple switch that turns it on or off.(Encapsulation). Most, if not all, lamps can be plugged into all outlets and use any type of light bulb. (Polymorphism) Lamps are all built similarly but some can have differences. (Inheritance) I think that the benefit of Object oriented programming is having flexible code. You can create an object and use it in other programs.

Modern JavaScript Development

The most I got from this lesson are the concepts of: Libraries, Modular JavaScript, and Package Manager.

Libraries:

The most interesting thing about libraries is that they are specialized for every use, for example:

  • Libraries for DOM manipulation:
    • jQuery
    • midori
    • MooTools
  • Libraries for Graphical/visualization
    • Babylon.js
    • Chart.js
    • D3.js, I actually found this video that explains in less than 3 minutes what is D3.js, and how to use it:

And there are libraries that are specialized in GUI (Graphical user interface) and widget related, Pure JavaScript/Ajax, Template systems, Unit testing and Web-application related (MVC, MVVM).

Modular JavaScript:

Modular JavaScript is basically the use of JavaScript is separate files to make the code more readable and usable.

Package manager:

In simple words, a package manager is a tool that allows users to install, remove, upgrade, configure and manage software packages on an operating system. The package manager can be a graphical application like a software center or a command line tool like apt-get or pacman. .

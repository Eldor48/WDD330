The call() method

  • The call() method can be used to set the value of this inside a function to an object that is provided as the first argument. Here is an example that I created: 
var obj = {
        a: 1,
        b: 2,
        c: 3
    };
    
    // The value of this is the global object
    console.log(this);
    console.log(this.a);
    console.log(this.c);
    
    // The value of this is obj
    obj.func = function() {
        console.log(this);
        console.log(this.a);
        console.log(this.c);
    };
    
    obj.func();
    
    // The value of this inside the function is set to obj2
    var obj2 = {
        a: 4,
        b: 5,
        c: 6
    };
    
    obj.func.call(obj2);

Recursive Functions

A recursive function is one that invokes itself until a certain condition is met. It’s a useful tool to use when iterative processes are involved.A classic example of recursion is the factorial function. If we want to find the factorial of a number we multiply it by every number smaller than it, so the factorial of 4 = 4 * 3 * 2 * 1 = 24.

There are two key parts to a recursive function:

The base case – the most basic part of the function which can be computed without recursion.

– the most basic part of the function which can be computed without recursion. The recursive case – the part of the function which calls itself to work out the solution.

Functional Programming

  • In my own words functional programming is a programming paradigm that emphasizes the use of functions. This programming style is often used in conjunction with languages that support first-class functions, such as Clojure, Scala, and Erlang.
  • In recent years, functional programming has gained popularity due to its ability to improve code readability and reduce the amount of code needed to complete a task. JavaScript is particularly well suited for functional programming due to its support for first-class functions. This means that functions can be passed as arguments, returned from other functions, and used anonymously. Closures are also a fundamental element of functional programming in JavaScript.

Currying

  • Functional programming usually encourages developers to write declarative code. This means that you should state what you want, and functions should calculate the result.
  • It also means that you should never change the state of your program. This can be accomplished by using immutable objects, or by using pure functions. Pure functions are ones that:
  • Always return the same result if the same arguments are passed in
  • Do not have side effects
  • Do not depend on the state of the program

Pure function

  • A function is called a pure function if it satisfies the following two conditions.
  • The function always produces the same output when given the same input. The function does not produce any side effects.

AJAX

  • AJAX is a web development technique for creating interactive web applications. AJAX allows web pages to be updated asynchronously by exchanging small amounts of data with the server behind the scenes. This means that it is possible to update parts of a web page, without reloading the whole page.
